ZR-UX Operation Specification

1. Scope & Positioning

ZR-UX is an interface layer that assists cognitive processing without replacing judgment. It allows clarification, stabilization, and completion only within explicitly defined bounds. ZR-origin remains a closed object; this specification applies only to ZR-UX.

2. Layer Definitions

L1. Translation / Unpacking

Purpose: Expand compressed ZR structures into accessible forms.

Allowed: term clarification, minimal sequencing

Forbidden: judgment, recommendation, responsibility transfer

L2. Stabilization / Cognitive Load Management

Purpose: Reduce overload and align perception.

Allowed: lists, diagrams, re-grouping

Limit: stop at cause separation

L3. Boundary Softening

Purpose: Present responsibility boundaries as optional decision points.

Allowed: soft boundary marking

Forbidden: option ranking, guidance

L4. Attribution / Reference

Purpose: Prevent misattribution of judgment or authority.

Allowed: single anchor reference

Forbidden: repetitive citation

L5. Completion / Resolution (UX-only)

Purpose: Provide closure for reporting or explanation contexts.

Allowed only when explicitly requested

Never active in ZR-origin

3. Activation Signals (Detection)

Transition to choice or judgment

Attribution

Structuring third-party material

Risk of advice misreading

Completion

Explicit request for summary, conclusion, or report

4. Signal Weights (Priority)

Priority order (highest first):

Responsibility Boundary

Cognitive Overload

Understanding Failure

Misattribution Risk

Completion Request

Higher-priority signals suppress lower ones.

5. Conflict Resolution (Tiebreaker)

Boundary overrides all other layers

Overload blocks completion

Translation stops when boundary appears

Structure always precedes attribution

If in doubt, output less
